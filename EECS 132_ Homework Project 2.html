Part I: Programming (60% of your grade will be correctness of the following)

Try to keep your code as simple as possible.

Create a class called HW2 that contains the following methods:

  1. everykToTheNthChar takes a String and an int as input and returns a String. Let us call the second input k (but you can name it something else). The method returns a String that contains the 1st character of the input, the kth characther of the input, the k^2th character of the input, the k^3th character of the input, etc. Return the input string if k is smaller than 2. 

    > HW2.everykToTheNthChar("1234567890", 2)
"1248"
> HW2.everykToTheNthChar("1234567890", 3)
"139"
> HW2.everykToTheNthChar("Case Western", 3)
"Cst"

  2. containsSubSequence takes two Strings as input and returns a boolean: Returns true if the first input string contains all the characters of the second input string, in order, but not necessarily consecutively. 

    > HW2.containsSubSequence("abracadabra", "abcd")
true
> HW2.containsSubSequence("abracadabra", "abdc")
false

  3. subtract takes two Strings and returns a String:
    The output string should be the same as the first input string except that you should remove the characters of the second string as follows: remove the first occurrence of the first character, the first occurrence of the second character that appears after the removed character, the first occurrence of the third character that appears after the first two removed characters, etc. If there is no such character to remove, then nothing further is removed. 

    HW2.subtract("gfedcbabcdefg", "acfa")
"gfedcbbdeg"

  4. doubleValue takes a String as input and returns a double:
    The input should contain only digits, up to one period '.' and may contain one dash, but only as the first character '-'. The method should return the double value that input string represents. The input may have any number of leading or trailing zeros. If the number is not in the correct format (more than one period, the dash is not the first character, or any other non-digit character), then report an error by using the statement throw new NumberFormatException(). Note that there may be very small unavoidable errors due to converting the decimal notation of the string to the binary of the double. 

    > HW2.doubleValue("0034.143003")
34.143003
> HW2.doubleValue("0034.143.003")
java.lang.NumberFormatException

  5. everyNthCharOfWord takes a String and an int as input and returns a String:
    If we call the input int value n, the output should contain the nth character of each word of the input string, in order. If the word does not have n characters, a space should be used for the character. A word is defined as a consecutive sequence of English letters. 

    > HW2.everyNthCharOfWord("Hello everyone!  This is terrific fun!", 2)
"evhseu"
> HW2.everyNthCharOfWord("Hello everyone!  This is terrific fun!", 4)
"lrs r "

  6. flipWords takes a String as input and returns a String:
    Each word of the input string is reversed in the output string, though the order of the words stays the same. A word is defined as a consecutive sequence of English letters. 

    > HW2.flipWords("Seventy-six trombones in the big parade!!")
"ytneveS-xis senobmort ni eht gib edarap!!"

  7. Extra Credit: transpose: Takes a String as input and returns a String:
    The output string is ordered so that, if it is printed, the first word of the input is printed vertically in the first column. The second word appears vertical in the second column, and so forth. The total number of rows of text should equal the length of the longest word. Here, a word is either a sequence of consecutive English letters or a sequence of non-whitespace, non-letters. Part 2: Project Testing (20% of your project grade)

You need to write tests for each of the methods above. Your tests should be thorough. Your testing report should describe why your test cases sufficiently tested your code as well as the specific tests you ran and their results. If you are unable to complete a method above, you should still describe tests that would test the method had it been completed.

Hints for testing loops. Your tests need to, at the minimum cover the following cases:

  1. Test 0, test 1, test many: This means you have to test cases where the parameters, if they are integers, are 0, 1 or some value other than 1. If the parameters are strings, you have to test strings of length 0, 1, and more than 1. If the strings must contain certain data, you need to test cases where they contain 0, 1, and more than 1 of the desired data.
  2. Test first, last, and middle: In cases where you have to search in or modify a string, you need to test cases where the item to be found or modified is the first character of the string, the last character of the string, or somewhere in the middle of the string.

In summary, your tests should cover each of the above cases for each of the methods you complete in the homework. Your testing report should state what test 0, test 1, test many and test first, last, and middle mean for each of the homework methods and how your tests covered these cases.

JUnit: you will be learning about JUnit soon in a lab. JUnit will be required for future homeworks, and you are welcome to use it with this homework. If you choose to write JUnit tests for your code, your report should not list the specific tests and their results because the graders can run your JUnit tests. Instead, your report should only detail what test 0, test 1, test many and test first, last, and middle means for each method and why your JUnit tests cover those cases.
